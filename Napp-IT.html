Testons OmniOS :

Doc

  • Au reboot les identifiants sont : root , mot de passe vide

Réseau

DHCP

ipadm create-if e1000g0
ipadm create-addr -T dhcp e1000g0/v4
echo 'nameserver 192.168.1.1' >> /etc/resolv.conf
cp /etc/nsswitch.conf{,.bak}
cp /etc/nsswitch.{dns,conf}

Fixe

ipadm create-if e1000g0
ipadm create-addr -T static -a 192.168.1.10/24 e1000g0/v4static
route -p add default 192.168.1.1
echo 'nameserver 192.168.1.1' >> /etc/resolv.conf
cp /etc/nsswitch.conf{,.bak}
cp /etc/nsswitch.{dns,conf}

Mettre le pool de démarrage en miroir

[2]

Sur une VM VirtualBox dont les disques sont : 

  format
 c1t0d0
 c1t2d0
 c1t3d0

fdisk -B   c1t2d0p0
prtvtoc /dev/rdsk/c1t0d0s2 | fmthard -s - /dev/rdsk/c1t2d0s2
zpool attach -f rpool c1t0d0s0 c1t2d0s0
 zpool status
installgrub /boot/grub/stage1 /boot/grub/stage2 /dev/rdsk/c1t2d0s0

Et désormais le système boote sur le miroir. (Si le second disque est déclaré bootable dans le BIOS)
