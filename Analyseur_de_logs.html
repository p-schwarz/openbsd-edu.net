Soucis législatifs

Why Splunk

Pas libre, mais gratuit jusqu'à 500 Mo de log quotidiens. Splunk semble être la rolls des parseurs de logs. Encaisse sans broncher des quantités conséquentes de logs (big data z'avez-dit ?); aucun parser à écrire.

Why not Splunk

Mais attention ! 500Mo/j c'est une limite soft. Dépassez 3 fois et vous n'aurez plus accès à grand chose, il faudra passer à la case banque et SPlunk, c'est très bien, mais très cher!! Dommage!

Un agent à déployer sur les machines à surveiller. Cet agent parse les logs et les envoie vers le serveur Splunk. [2]

raspBian/Raspberry pi

http://apps.splunk.com/app/1611/ 

tar zxvf splunkforwarderforunix.tgz
sudo mv splunkforwarder /opt
sudo /opt/splunkforwarder/bin/splunk start
sudo /opt/splunkforwarder/bin/splunk  add forward-server ServeurSplunk:9997
sudo /opt/splunkforwarder/bin/splunk  enable boot-start
sudo /opt/splunkforwarder/bin/splunk  start



sudo /opt/splunkforwarder/bin/splunk  add monitor /var/log/auth.log
sudo /opt/splunkforwarder/bin/splunk  add monitor /var/log/auth.log
sudo /opt/splunkforwarder/bin/splunk  add monitor /var/log/syslog
sudo /opt/splunkforwarder/bin/splunk  add monitor /var/log/messages
sudo /opt/splunkforwarder/bin/splunk  add monitor /var/log/dmesg

Lancement

Les commandes sont self-explicites : 

/opt/splunkforwarder/bin/splunk  add forward-server ServeurSplunk:9997
/opt/splunkforwarder/bin/splunk  add monitor /var/log/squid/blocked.log
/opt/splunkforwarder/bin/splunk  enable boot-start
/opt/splunkforwarder/bin/splunk  start

Requêtes amusantes

Métadonnées des échanges

host="IP_du_srv_postfix" postfix ("to=" OR "from=")

Donne les échanges de mails. Pas le contenu des mails (heureusement), mais les (famous) métadonnées :Qui cause à qui!

Combien de mails par heure

host="IP" postfix ("to=" OR "from=") |rex field=_raw "From: (?<from>.*) To: (?<to>.*)" | chart count(from) by date_hour

Usages

Gérer son compte en banque

[6]

  • Sur votre site de banque en ligne télécharger le .dat au format tableur.
  • Un coup de perl/sed:Whatever :
perl -pi -e 's/\t/|/g' XXXXXX.dat 
Ajouter une ligne iinitiale qui contiendra les colonnes :

Date_operation|Numero_operation|Media_operation|Montant_operation|Tiers_operation|Ref_operation

  • Add data sur Splunk
Source type format : structured/psv
Break Type : Every line
Mode table
No line number
UTC +1
Comma separated field name : date_operation,montant,import_time,,numero_operation,field5,media,tiers

Attention, field_names doit être sur Auto

Enregistrez le format d'import et créez un nouvel index sandbox pour tester.

Si l'import vous semble correct, vous pouvez modifier l'index ultérieurement ( [7] ). (Settings/Indexes/sandbox;//) Mieux, vous pouvez refaire l'opération sur l'index default.

Sur un serveur Debian à monitorer

Recopier le certificat /etc/pki/tls/certs/logstash-forwarder.crt depuis le serveur logstash 

wget -O - http://packages.elasticsearch.org/GPG-KEY-elasticsearch |  apt-key add -
echo 'deb http://packages.elasticsearch.org/logstashforwarder/debian stable main' |  tee /etc/apt/sources.list.d/logstashforwarder.list
apt-get update
apt-get install logstash-forwarder

Et cas de souci, voir +bas, rubrique #Debug 

cd /etc/init.d/
wget --no-check-certificate https://raw.github.com/elasticsearch/logstash-forwarder/master/logstash-forwarder.init -O logstash-forwarder
chmod +x logstash-forwarder
update-rc.d logstash-forwarder defaults
mkdir -p /etc/pki/tls/certs
cp /tmp/logstash-forwarder.crt /etc/pki/tls/certs/
vi /etc/logstash-forwarder

{
 "network": {
   "servers": [ "IP_ELKS:5000" ],
   "timeout": 15,
   "ssl ca": "/etc/pki/tls/certs/logstash-forwarder.crt"
 },
 "files": [
   {
     "paths": [
       "/var/log/syslog",
       "/var/log/auth.log"
      ],
     "fields": { "type": "syslog" }
   }
  ]
}
