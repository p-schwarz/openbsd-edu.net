Configuration

Depuis Vsphere client (ex virtual center)

  • On s'enregistre

Ajout du support iSCSI

  • Configuration / Networking / Add networking / VMKernel /

On veut installer le pilote iSCSI sur une nouvelle carte physique =>

  • Create a virtual switch vmnic2
  • Port Group properties : Nom : VMKernel.san VLAN : 112
  • IP : 172.12.0.1/24
  • Creation du storage iSCSI
  • Ajout du service console sur la carte vmnic1
172.12.0.101/24


Une fois que tout cela est fait, on passe à la suite .

  • Ajouter LUN iSCSI TODO


  • Ajouter stockage
    • Onglet Configuration/Storage/Add Storage
    • Disk/LUN

Parti ! On va pouvoir commencer à jouer avec nos VM !


Images ISO

Il ne faut PAS utiliser les CDROM/DVDROM, mais des images ISO ! Non n'insistez pas, saimal

  1. Creer une image iso : 
dd if=/dev/cdrom of=OS.iso
  1. La copier sur clé USB
  2. Sur VSPhere Client :
  • Configuration
  • Storage
  • Choisir son datastore
  • Browse Datastore
  • Trouver le bon repertoire
  • Upload File / Folder
  1. VM / Edit Settings
  • Le CD/DVD doit être connecté au boot
  • Choisir Datastore ISO File

Les commandes utiles

Gestion des VM

  • Page utile
  • vmware-cmd -l : Liste les VM disponibles
/vmfs/volumes/UID../SRSS1/SRSS1.vmx
/vmfs/volumes/UID.../Pericles-W2003R2/Pericles-W2003R2.vmx
/vmfs/volumes/UID../Faraday-Radius/Faraday-Radius.vmx
  • vmware-cmd /vmfs/volumes/UID.../SRSS1/SRSS1.vmx getstate  : Donne l'état de la VM
 getstate() = on
  • vmware-cmd /vmfs/volumes/UID.../SRSS1/SRSS1.vmx getuptime  : Donne l'uptime en secondes de la VM
  • Un script qui fait tout cela
  • ' vmware-cmd vmfs/volume/vmfslabel/VMName/VMName.vmx suspend  :Suspend
