ZenBook UX31A

Sans aucun doute la plus belle machine que j'ai jamais eue.

Minuscule, silencieuse, surpuissante.

Juste l'écran dont la résolution est un poil élevée pour du 13 pouces.

Avant toute chose, une sauvegarde de tout le disque avec clonezilla live est indispensable !

Pas juste les partitions, mais le MBR/UEFI également !

  • Hint : Le mdp est live

C'est vraiment important pour ne pas avoir à pleurer/renvoyer au SAV (ce qui est assez semblable..)

Réseau

Edit : Le dongle USB est 2.0 alors que les ports sont en 3.0 ! Résultat, pas de Gigabit avec le dongle de base!!

En achetant un vrai adaptateur USB3.0, on arrive au Gb. Genre Startech.com

Il faut juste compiler le module du chipset: 

wget http://www.asix.com.tw/FrootAttach/driver/AX88179_178A_LINUX_DRIVER_v1.5.0_SOURCE.tar.bz2
bunzip2 AX88179_178A_LINUX_DRIVER_v1.5.0_SOURCE.tar.bz2 
tar xf AX88179_178A_LINUX_DRIVER_v1.5.0_SOURCE.tar 
cd AX88179_178A_LINUX_DRIVER_v1.5.0_SOURCE
make
make install
insmod ax88179_178a.ko

EFI

Tout d'abord il faut trouver la partition EFI indispensable au démarrage.

Partitions

  • La partition sda1 id 1C (Hidden W95 FAT32 LBA) de taille 26 Go est la partition système Windows ??
  • La partition sda2 id 27 (Hidden NTFS WinRE) de taille 100Mo est en plus bootable.
  • La partition sda3 id 07 (HPFS/NTS/exFAT) de taille 107Go est
  • La partition sda4 id 0F (W95 Ext'd LBA) de taille 116Go est
  • La partition sda5 id 84 de taille 4Go est le suspend to disk de Windows
  • La partition sda6 id 07 (HPFS/NTS/exFAT) de taille 112Go est


Edit: En fait, il n'y a aucun souci ! Il suffit de détruire toutes les partitions, et de recréer au moins une partition de type EFI, installée dans /boot/efi

Installation de Ubuntu 12.04

SSD

En fait, on se calme avant de tout virer, la machine démarre sur de l'UEFI et plus du BIOS... Grosse galère si vous virez toutes les partitions...

Quelques idées  :[2] et Donc :

  • fdisk est trop vieux pour comprendre le format UEFI
  • parted lui, fonctionne

Il faut recréer la partition EFI!

  • Lancer une install Ubuntu 12.04, au hasard.
  • Créer une partition FAT16, flag bootable n'importe où sur le disque (sda4, par ex.). gparted ou parted seront vos (seuls) amis.
  • Suivre le tuto [3]
sudo mkdir /mnt/temp
sudo mount /dev/sda4 /mnt/temp
for i in /dev /dev/pts /proc /sys; do sudo mount -B $i /mnt/temp$i;  done
sudo chroot /mnt/temp
mkdir /boot/efi
blkid |grep vfat

Notez l'UUID, en respectant la casse!

Recopiez cet identifiant dans /etc/fstab

UUID=XXXX-XXXX /boot/efi vfat defaults 0 1

mount /boot/efi doit fonctionner

Si vous êtes en cours d'installation, le nom d'hôte va changer en xxxxxx 

echo "127.0.0.1  xxxxx" >> /etc/hosts
apt-get install grub-efi
grub-install
cp -r /boot/efi/EFI/ubuntu /boot/efi/EFI/xxxxx
exit
reboot
croisez les doigts
